# Adding voice narration to an HTML demo

When the demo needs a TTS voice-over that **stays aligned** to the on-screen choreography without manually re-timing every edit. This reference covers the toolchain (`generate-tts.py` → `transcribe.py` → `dump-timestamps.py`), the "audio as master clock" playback pattern, and the gotchas you only learn by hitting them.

If your demo only needs visuals — skip this entirely. The patterns in `SKILL.md` work standalone.

## The core idea: audio is the master clock

The naive approach — generate audio, hope it lines up with the timeline — drifts every time you edit a script. The robust approach:

1. **Write narration as plain text** (one file per scene)
2. **Generate TTS audio** via OpenAI `gpt-4o-mini-tts` or Alibaba DashScope `qwen3-tts-flash`
3. **Transcribe the generated audio** with Whisper to get word-level timestamps
4. **Define checkpoint anchors** — an array of `[audioTime, sceneTime]` pairs that tie spoken phrases to scene-time beats
5. **Drive the visual timeline from audio**: a `requestAnimationFrame` loop reads `audio.currentTime`, linearly interpolates between checkpoint pairs, and seeks the GSAP timeline

Non-uniform pacing falls out for free. Segments of audio with dense narration play faster on the scene side; quieter beats give visuals room to breathe.

## Workflow

### 1. Write scripts

Each `.txt` file is one scene. Filename (sans `.txt`) is the scene id and becomes the `<audio>` element's id (e.g. `audio-intro`).

Structural template that works well:

| Section | Duration | Purpose |
|---|---|---|
| Intro | 4–10 s | "Meet [persona] — [role / context]. [Stakes line.]" |
| Action | 25–40 s | Walk through what happens, naming the moments the visuals show |
| Recap | 6–15 s | Summarize what the scene demonstrated — a concise close |

Aim for **~100–140 words per scene** at default TTS speed (lands at 50–65 s of audio). Longer overruns the visuals; shorter leaves dead air at the end.

The recap section is where you should explicitly name the architectural axes / system components engaged in the scene. The viewer hears "Skills did X, Memory carried Y, Runtime did Z" exactly when the visual axis pills are all lit. Double-encoding makes the vocabulary land.

### 2. Generate TTS audio

`generate-tts.py` supports two backends; pick by `--backend`:

```bash
# OpenAI (default voice: ash)
python3 scripts/generate-tts.py \
  --scripts-dir audio/scripts --out-dir audio \
  --backend openai --voice ash

# Alibaba DashScope / Qwen-TTS (default voice: Cherry)
python3 scripts/generate-tts.py \
  --scripts-dir audio/scripts --out-dir audio \
  --backend qwen --voice Cherry --region cn
```

#### Picking a backend

The script's default is `--backend auto`:
- If only `OPENAI_API_KEY` is present in env or `.env` → use OpenAI.
- If only `DASHSCOPE_API_KEY` is present → use DashScope.
- If both are present, the script prompts interactively (in non-TTY contexts it exits with a clear message asking for `--backend`).

When unsure which the user wants, **ask first** rather than guessing. The two backends have meaningfully different voice character, and the API keys often correspond to billing accounts the user picks for a reason. Useful prompts to gather context: "what region are you in?" / "do you have an OpenAI or DashScope key, or both?" / "any voice character preference (English-leaning vs. CJK-leaning, neutral vs. warm)?"

#### OpenAI voices (`gpt-4o-mini-tts`)

| Voice | Character | Good for |
|---|---|---|
| `ash` | confident, articulate, modern | tech / product demos (default pick) |
| `sage` | measured, mature | longer-form explainers |
| `verse` | expressive, narrative | story-mode pieces |
| `echo`, `alloy` | neutral | when you don't want a strong personality |
| `onyx` | deep, authoritative | enterprise / serious |
| `coral`, `nova`, `shimmer` | friendlier, lighter | consumer / playful |
| `ballad`, `fable` | narrative / storybook | brand-story pieces |

OpenAI also accepts `--speed 0.85–1.15` and `--instructions "tone description"`. The instructions parameter shapes prosody meaningfully — keep it short and concrete ("calm technical narrator with deliberate pacing" > "engaging and dynamic voice").

#### Qwen / DashScope voices (`qwen3-tts-flash`)

40+ voices including English-fluent (`Cherry`, `Ethan`, `Serena`, `Ryan`, `Jennifer`) and regional CJK voices (`Jada` for Shanghainese, `Dylan` for Beijing, plus Cantonese, Sichuan, Korean, Japanese, etc.). DashScope auto-detects language from the text; we set `language_type` heuristically by scanning for CJK characters.

Qwen-TTS is the better choice when narration mixes English with significant Chinese spans — its bilingual prosody is more natural than OpenAI's. For pure English, OpenAI's `ash` voice is hard to beat. Try both on a 2-sentence sample before committing to one for a full demo.

#### Regional endpoints (Qwen only)

DashScope uses region-specific API keys and base URLs:

| `--region` | Endpoint | Use when |
|---|---|---|
| `cn` (default) | `https://dashscope.aliyuncs.com` | China account / China-issued API key |
| `intl` | `https://dashscope-intl.aliyuncs.com` | Singapore account / International key |
| `us` | `https://dashscope-us.aliyuncs.com` | US (Virginia) account / US key |

**Keys are not interchangeable across regions** — using a China key against the Singapore endpoint will return an auth error. The script defaults to `cn` but honors `DASHSCOPE_REGION` from the env. If you don't know which region the user's key is for, ask them — the console URL they used to create it is the giveaway.

#### Output format

Both backends produce `.mp3` by default. OpenAI returns mp3 directly; Qwen returns wav and the script converts via ffmpeg (`libmp3lame -qscale:a 2`). Pass `--out-format wav` to skip conversion if you want the original Qwen WAV (24-bit, higher quality but ~5× larger).

#### Re-generation tips

Re-generate only what changed:
```bash
python3 scripts/generate-tts.py --only intro,outro
```

If audio is consistently too fast/slow, regenerate with `--speed 0.95` or `--speed 1.05` (OpenAI). Qwen-TTS doesn't expose speed — adjust the text instead (shorter sentences read faster).

### 3. Transcribe for timestamps

```bash
python3 scripts/transcribe.py --audio-dir audio
```

Writes `{scene}.json` next to each mp3 with `words`, `segments`, `text`, and `duration`.

### 4. Find anchor times

```bash
python3 scripts/dump-timestamps.py --audio-dir audio
```

This prints phrase-level chunks with start–end times. Pick 6–12 anchor phrases per scene — moments where the spoken word should land on a visual beat. Example output:

```
══════ intro (43.70s) ══════
   0.00- 2.68  3 a m pager duty fires
   3.46- 5.44  Payment service latency just tripled
   6.20- 9.72  Ops is already on it Skills pull metrics
  ...
```

### 5. Define scene checkpoints

In the HTML's scene config:

```js
scenes.intro = {
  sceneEnd: 30.0,                 // GSAP timeline's built duration
  checkpoints: [
    [0.00,  0.0],                 // intro narration → scene idle
    [7.50,  0.0],                 // hold during context setup
    [8.54,  1.5],                 // first action: "pager fires" → PD card slides in
    [16.54, 4.0],                 // "Skills pull metrics" → triage starts
    [28.62, 12.0],                // "Runtime drafts hypothesis"
    [32.92, 17.0],                // "He taps once" → approval beat
    [43.94, 31.0],                // "Five axes did the work" → closing strip
    [61.51, 34.0],                // end of audio → scene end
  ],
  buildTimeline() { /* paused timeline */ },
};
```

Rules of thumb:
- **First row** always `[0.00, 0.0]`
- **Last row** maps end-of-audio to end-of-scene
- During the **intro period**, hold scene-time at the first action's pre-state (don't play visuals while you're saying "meet so-and-so")
- During the **outro / axis recap**, slowly walk scene-time toward `sceneEnd` so the closing card stays visible across the spoken summary
- Aim for ~6–12 anchors; between any two anchors, scene-time interpolates linearly with audio-time

## Wire audio-as-master playback

Six pieces in the HTML.

**a.** One `<audio>` element per scene:
```html
<audio id="audio-intro" preload="auto" src="./audio/intro.mp3"></audio>
<audio id="audio-act-1" preload="auto" src="./audio/act-1.mp3"></audio>
```

**b.** Checkpoint interpolation:
```js
function audioToScene(audioT, scene) {
  const cps = scene && scene.checkpoints;
  if (!cps || !cps.length) return audioT;
  if (audioT <= cps[0][0]) return cps[0][1];
  for (let i = 0; i < cps.length - 1; i++) {
    const [a0, s0] = cps[i];
    const [a1, s1] = cps[i + 1];
    if (audioT <= a1) {
      const span = a1 - a0;
      const frac = span > 0 ? (audioT - a0) / span : 0;
      return s0 + frac * (s1 - s0);
    }
  }
  return cps[cps.length - 1][1];
}
```

**c.** rAF drive loop — audio drives the GSAP timeline via seek:
```js
let driving = false;
function driveLoop() {
  if (!driving || !currentAudio || !currentTl) return;
  if (currentAudio.paused) { driving = false; return; }
  const sceneT = audioToScene(currentAudio.currentTime, scenes[activeSceneId]);
  currentTl.time(sceneT);
  requestAnimationFrame(driveLoop);
}
function startDrive() {
  if (driving) return;
  driving = true;
  requestAnimationFrame(driveLoop);
}
function stopDrive() { driving = false; }
```

**d.** Build the GSAP timeline **paused** (`gsap.timeline({ paused: true })`) — you'll seek it via `currentTl.time()`, not let it auto-play.

**e.** Transport operates on audio:
```js
// Play:    audio.play() → startDrive()
// Pause:   audio.pause() → stopDrive()
// Restart: scene.reset(); audio.currentTime = 0; audio.play()
// Scrub:   audio.pause(); audio.currentTime = frac * audio.duration; sync timeline once
// Mute:    audio.muted = true (audio keeps playing silently so scene keeps moving)
```

**f.** Inverse mapping for scrubber tick marks — each beat's marker shown at its audio-time position, not scene-time:
```js
function sceneToAudio(sceneT, scene) {
  const cps = scene && scene.checkpoints;
  if (!cps || !cps.length) return sceneT;
  if (sceneT <= cps[0][1]) return cps[0][0];
  for (let i = 0; i < cps.length - 1; i++) {
    const [a0, s0] = cps[i];
    const [a1, s1] = cps[i + 1];
    if (sceneT <= s1) {
      const span = s1 - s0;
      const frac = span > 0 ? (sceneT - s0) / span : 0;
      return a0 + frac * (a1 - a0);
    }
  }
  return cps[cps.length - 1][0];
}
```

## Smooth entry / exit

- **Fade-in** narration at scene load: set `audio.volume = 0`, call `audio.play()`, tween volume to 1 over ~0.4 s
- **Pause/resume**: brief fade-out (0.2 s) on pause, brief fade-in on resume — avoids audio clicks
- **Scene switch**: stop previous audio immediately, swap `currentAudio`, fade in new — silence between is fine and reads as a scene break
- **Mute**: prefer `audio.muted = true` over pausing — keeps audio decoding so the scene keeps advancing in lock-step

## Audio-specific gotchas

**Autoplay policy** is the big one. Browsers reject `audio.play()` without a prior user gesture. The first scene after a fresh page load may play silently. Once the user clicks anywhere — play button, scrubber, persona card — `audio.play()` works for the rest of the session. **Don't gate visuals on the audio promise** — let the timeline render at t=0 even if audio is blocked, and the play button kicks things off.

**Scrubbing re-fires `.call()` callbacks**. GSAP fires `.call()` each time the playhead crosses them going forward. Rapidly scrubbing back-and-forth re-triggers any sound effects or one-shot animations attached as callbacks. Acceptable for a demo; if you really need single-fire, gate inside the callback with a "fired this run" flag and clear it on restart.

**Audio loading lag**. `preload="auto"` is a hint, not a guarantee. Start the drive loop conditionally on `isFinite(audio.duration)`. If `audio.duration` is NaN, fall back to scene's own `sceneEnd` for the time-elapsed display.

**TTS isn't deterministic across runs**. Even with the same script, voice, and speed, both OpenAI and Qwen-TTS produce slightly different timings each call. After a regeneration, always re-run `transcribe.py` and re-pick checkpoint anchors. Word-level timestamps drift by ~0.1–0.4 s from what you'd say "feels right" — test by ear and nudge anchors ±0.3 s as needed.

**DashScope URLs expire in 24 h** but the script downloads immediately so this only matters if you save the response JSON and try to download later. Just re-run the script if you need to regenerate.

**DashScope region/key mismatch returns 401**. The error message "InvalidApiKey" with a 401 status when calling `dashscope.aliyuncs.com` from a Singapore key (or vice versa) is the common case. Always set `--region` (or `DASHSCOPE_REGION`) to match the key's region — don't assume `cn`.

**Whisper transcripts sometimes mangle proper nouns** (a brand name becomes a similar-sounding word). The timestamps are still accurate; just match against the *spoken* phrase ("OpenClaw" not "OpenPaw") when finding anchors.

**Word-by-word vs phrase-by-phrase anchoring**. Word-level is too granular and noisy. Pick anchor *phrases* — usually the first word of a key sentence. Diminishing returns past ~12 anchors per scene.

**Don't mix `timeScale()` and audio-driven seek**. If the timeline has a non-1 timeScale, `tl.time()` still uses timeline-time but real-time pacing diverges — you'll get drift. Always build the timeline with `timeScale = 1` and rely on checkpoint stretching for pacing.

## Voice / instruction tuning

The `--instructions` parameter on `gpt-4o-mini-tts` shapes tone meaningfully. Useful patterns:

- **Confident product demo**: *"Calm, confident, professional product-demo narration. Measured pacing with natural pauses between sentences."*
- **Warm explainer**: *"Friendly, warm narration like explaining something to a thoughtful colleague. Slow down on technical terms."*
- **Cinematic / dramatic**: *"Cinematic narrator voice. Build tension through measured pacing. Italicize key phrases with emphasis."*
- **Mixed languages**: *"Speak Chinese phrases with natural Mandarin pronunciation; tech terms stay English."*

Qwen-TTS doesn't accept free-form instructions on the standard `qwen3-tts-flash` model — pick the voice that fits and shape pacing via the text itself (shorter sentences, comma rhythm). The `qwen3-tts-instruct-flash` model accepts an `instruct` field but is slower and not used by `generate-tts.py` by default; consider it only if voice-character control matters more than throughput.

Iterate fast on either backend: regenerate one scene with `--only sceneName` to compare voices before committing.

